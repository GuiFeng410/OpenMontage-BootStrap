# 第一步：下载必要文件与运行时

目标：新机器上具备「能跑 doctor / media /（可选）providers-tts」的代码与依赖，**尚未**配置 OpenClaw。

完成后应得到：仓库根路径 `<OPENMONTAGE_REPO_ROOT>`、可写沙箱 `<PROJECTS_DIR>`、可用的 Python venv、Piper 中文模型、Remotion `node_modules`、FFmpeg。

---

## 1.1 下载 / 克隆代码仓

- [ ] `git clone` 本仓库到目标机（推荐），或拷贝完整工作副本  
- [ ] 检出含 P2 TTS 的提交（例如 `17b7629` 或更新的 `main`）  
- [ ] 记下根目录为 `<OPENMONTAGE_REPO_ROOT>`

### 最小目录（缺一不可）

| 路径 | 用途 |
|------|------|
| `openmontage/mcp/` | doctor / media / providers_tts |
| `openmontage/skills/` | OpenClaw Skill Pack |
| `tools/` | Piper、diagram、compose 等 BaseTool |
| `lib/` | checkpoint、pipeline 加载 |
| `schemas/` | artifact 校验 |
| `pipeline_defs/` | 尤其 `animated-explainer.yaml` |
| `skills/pipelines/explainer/` | 各阶段 director |
| `skills/meta/` | Rule Zero、atelier 等 |
| `remotion-composer/` | Remotion 渲染工程 |
| `requirements.txt` | Python 依赖 |
| `scripts/scaffold_atelier_project.py` | atelier 脚手架（强烈建议） |

### 不要从旧机器直接拷

- [ ] 不要拷 `.venv/`（目标机重建）  
- [ ] 不要拷 `remotion-composer/node_modules/`（目标机 `npm install`）  
- [ ] 不要拷本机 `.env` / API Key（密钥在第二步按需配置）  
- [ ] `projects/` 旧成片可选，非必须

---

## 1.2 Python 环境

在 `<OPENMONTAGE_REPO_ROOT>`：

- [ ] 安装 Python 3.10+  
- [ ] 创建并激活 venv（示例）：

```powershell
cd <OPENMONTAGE_REPO_ROOT>
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -U pip
python -m pip install -r requirements.txt
```

- [ ] 自检导入：

```powershell
python -c "import openmontage.mcp.doctor; import openmontage.mcp.media; import openmontage.mcp.providers_tts; print('ok')"
```

---

## 1.3 Piper（零 Key TTS，最小出片必装）

- [ ] 确保 `piper-tts` 已随 `requirements.txt` 安装（或 PATH 中有 `piper`）  
- [ ] 下载中文模型，例如 `zh_CN-huayan-medium`，放到某目录 `<PIPER_MODEL_DIR>`  
- [ ] 准备环境变量（可先写进系统 / 用户环境，第二步再写进 MCP `env`）：

| 变量 | 示例值 |
|------|--------|
| `PIPER_MODEL_DIR` | 模型所在目录 |
| `OPENMONTAGE_PIPER_MODEL` | `zh_CN-huayan-medium` |

---

## 1.4 Node.js + Remotion

- [ ] 安装 Node.js 18+  
- [ ] 安装 Remotion 依赖：

```powershell
cd <OPENMONTAGE_REPO_ROOT>\remotion-composer
npm install
```

- [ ] 确认 `remotion-composer\node_modules` 存在

---

## 1.5 FFmpeg 与中文字体

- [ ] `ffmpeg`、`ffprobe` 在 PATH（`ffmpeg -version` 能跑）  
- [ ] 系统至少一套可渲染中文的字体（Windows 常见：微软雅黑）

---

## 1.6 项目沙箱目录

- [ ] 创建可写目录，例如：  
  - 仓库内：`<OPENMONTAGE_REPO_ROOT>\projects`  
  - 或独立：`C:\Users\<you>\OpenMontageProjects`  
- [ ] 记下为 `<PROJECTS_DIR>`  
- [ ] 准备环境变量：

| 变量 | 说明 |
|------|------|
| `OPENMONTAGE_PROJECTS_DIR` | 必须指向 `<PROJECTS_DIR>` |
| `PYTHONUTF8` | 建议 `1` |
| `OPENMONTAGE_P1_ALLOW_WRITES` | 出片时 `true`（第二步 Production Agent 再固化） |

本机快速探针（venv 已激活）：

```powershell
$env:PYTHONUTF8 = "1"
$env:OPENMONTAGE_PROJECTS_DIR = "<PROJECTS_DIR>"
$env:OPENMONTAGE_P1_ALLOW_WRITES = "true"
$env:PIPER_MODEL_DIR = "<PIPER_MODEL_DIR>"
$env:OPENMONTAGE_PIPER_MODEL = "zh_CN-huayan-medium"
python -c "from openmontage.mcp.doctor.tools import run_doctor; d=run_doctor(); print('can_produce=', d.get('can_produce_video_now')); print(d.get('next_install_for_p1'))"
```

- [ ] 争取 `can_produce_video_now == True`（若仍为 False，按打印的 `next_install_for_p1` 补齐）

可选测试：

```powershell
python -m pytest tests/mcp_p0 tests/mcp_p1 tests/mcp_p2 -q
```

## 1.7 （可选）付费 TTS 所需

最小出片**不需要**付费 Key。若新机也要用高级 TTS：

- [ ] 至少一个云 TTS API Key（OpenAI / ElevenLabs / DashScope / 豆包等）  
- [ ] 建议同时准备：`OPENMONTAGE_MAX_COST_USD`、`OPENMONTAGE_ALLOWED_PROVIDERS`  
- [ ] Key 只放环境变量或 MCP `env`，不要写进 Skill / 不要提交进 git
