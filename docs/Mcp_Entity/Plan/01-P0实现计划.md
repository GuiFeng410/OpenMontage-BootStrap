## 0. 一句话目标

交付一套可在 **OpenClaw 先跑通、再迁 eClaw** 的 P0 包：新人装上后，对 Agent 说「我想做视频」即可在约 60 秒内得到**环境诊断 + 能力分级 + 恰好 3 条可执行提示词**，**不出片、不调付费 API**。

## 4. MCP-A：`openmontage-doctor` 规格

### 4.1 传输与注册

```json
{
  "mcp": {
    "servers": {
      "openmontage-doctor": {
        "command": "python",
        "args": ["-m", "openmontage_mcp.doctor"],
        "cwd": "<OPENMONTAGE_REPO_ROOT>",
        "env": {
          "OPENMONTAGE_PROJECTS_DIR": "<PROJECTS_DIR>",
          "PYTHONUTF8": "1"
        },
        "toolFilter": {
          "include": [
            "doctor",
            "provider_menu_summary",
            "list_pipelines",
            "list_projects",
            "get_project_state",
            "get_next_stage",
            "validate_artifact",
            "validate_checkpoint",
            "estimate_cost",
            "init_project",
            "open_backlot"
          ],
          "exclude": []
        }
      }
    }
  }
}
```

注册命令（OpenClaw）：

```bash
openclaw mcp add openmontage-doctor \
  --command python \
  --arg -m \
  --arg openmontage_mcp.doctor \
  --cwd "<OPENMONTAGE_REPO_ROOT>" \
  --env OPENMONTAGE_PROJECTS_DIR=<PROJECTS_DIR> \
  --env PYTHONUTF8=1 \
  --include "doctor,provider_menu_summary,list_pipelines,list_projects,get_project_state,get_next_stage,validate_artifact,validate_checkpoint,estimate_cost,init_project,open_backlot"

openclaw mcp doctor openmontage-doctor --probe
```

eClaw：优先尝试等价 UI/导入；若字段不同，记入差异表，不改工具名。

### 4.2 工具契约（实现清单）

| Tool | 委托 / 逻辑 | 副作用 | 沙箱规则 |
|------|-------------|--------|----------|
| `doctor` | 查 `python`/`node`/`ffmpeg`/`ffprobe`/`npx`；Piper；Remotion `node_modules`；可选 HyperFrames；`registry.discover()` 摘要 | 无 | 只读 |
| `provider_menu_summary` | `registry.provider_menu_summary()` | 无 | 只读 |
| `list_pipelines` | 扫描已安装 pack 或 `pipeline_defs/*.yaml`（标注是否仅「文件存在」还是「Skill Pack 已装」） | 无 | 只读 |
| `list_projects` | `OPENMONTAGE_PROJECTS_DIR` 下 `project.json` | 无 | 只读该根 |
| `get_project_state` | 读 marker + checkpoints | 无 | 项目必须在根内 |
| `get_next_stage` | `lib.checkpoint.get_next_stage` | 无 | 同上 |
| `validate_artifact` | jsonschema + `schemas/artifacts` | 无 | 路径必须在根内 |
| `validate_checkpoint` | checkpoint schema | 无 | 同上 |
| `estimate_cost` | 解析 tool name → `BaseTool.estimate_cost`；未知工具报错 | 无 | 无执行 |
| `init_project` | `lib.checkpoint.init_project` | **创建目录** | 仅允许写到 `PROJECTS_DIR/<id>` |
| `open_backlot` | 尝试 `python -m backlot open [id]` | 可选起本地服务 | 失败返回指引，不抛致命 |

统一响应 envelope（建议）：

```json
{
  "ok": true,
  "contract_version": "0.1.0",
  "openmontage_version": "<git describe or package version>",
  "data": {},
  "warnings": [],
  "fix": null
}
```

### 4.3 `doctor` 输出形状（白话由 Skill 生成）

```json
{
  "tier": "zero-key | starter | standard | full | unknown",
  "platform": "win32|darwin|linux",
  "binaries": {
    "python": {"ok": true, "version": "...", "path": "..."},
    "node": {"ok": false, "fix": "not found", "fix_hint": "..."},
    "ffmpeg": {},
    "ffprobe": {},
    "npx": {}
  },
  "runtimes": {
    "remotion": {"ok": false, "detail": "remotion-composer/node_modules missing"},
    "hyperframes": {"ok": false, "detail": "..."},
    "piper": {"ok": false, "detail": "model missing"}
  },
  "registry": {
    "tool_count": 101,
    "available_count": 42,
    "by_capability_top": []
  },
  "quick_unlocks": [
    {"what": "Piper voice model", "commands": {"win32": ["..."], "darwin": [], "linux": []}}
  ],
  "hardware_unlocks": [],
  "installed_skill_packs": ["openmontage-router"],
  "can_produce_video_now": false,
  "next_install_for_p1": ["piper-tts", "remotion-composer npm", "ffmpeg"]
}
```

规则：

- **禁止**硬编码「共有 N 个工具」；`tool_count` 来自 registry。
- `can_produce_video_now`：P0 在未装 media pack 时恒为 `false`，即使本机已有 Remotion。
- 修复命令必须分 `win32` / `darwin` / `linux`；PowerShell 不用字面量 `%USERPROFILE%`。

### 4.4 路径沙箱

- 解析后的所有读写路径必须落在 `Path(OPENMONTAGE_PROJECTS_DIR).resolve()` 之下。
- 拒绝 `..`、符号链接逃逸、绝对路径越界。
- 未设置 `OPENMONTAGE_PROJECTS_DIR` 时：`doctor` / `provider_menu_summary` 仍可用；写类工具（`init_project`）返回明确错误。

### 4.5 MCP SDK 选型

| 选项 | 说明 | P0 建议 |
|------|------|---------|
| A. 官方 MCP Python SDK | stdio + tool schema 标准 | **默认** |
| B. 自研 JSON-RPC | 可控但易踩协议坑 | 不推荐 |

实现前锁定 SDK 版本并写入 `pyproject.toml`；用 `openclaw mcp probe` 作为兼容性门禁。

## 5. Skill Pack 规格

### 5.1 `skills_eclaw/openmontage-router/SKILL.md`

Frontmatter（按 Guide/02）：

```yaml
---
name: openmontage-router
description: Diagnose OpenMontage setup and give 3 machine-ready video starter prompts.
metadata:
  openclaw:
    requires:
      bins:
        - python
      env:
        - OPENMONTAGE_PROJECTS_DIR
      anyBins: []
    primaryEnv: OPENMONTAGE_PROJECTS_DIR
    envVars:
      - name: OPENMONTAGE_PROJECTS_DIR
        required: false
        description: Optional for pure diagnosis; required before init_project
      - name: OPENMONTAGE_REPO_ROOT
        required: false
        description: Monorepo root if MCP cwd is not set
    os:
      - win32
      - darwin
      - linux
    emoji: "🎬"
---
```

正文必须规定：

1. 模糊请求 → 调 `doctor` + `provider_menu_summary`（工具名以宿主投影为准，可能是 `openmontage-doctor__doctor`）。
2. 白话摘要；**不** dump 大 JSON。
3. 恰好 **3** 条提示词，且与 `can_produce_video_now` / tier 一致：
   - 若不能出片：3 条里至少 1 条是「先安装 P1 依赖」类，**禁止**假装零 Key 立刻成片。
   - 若本机已具备 P1 运行时但未装 media MCP：说明「机器可能够用，但当前只装了诊断包」。
4. 明确生产请求且未装 explainer pack → 给出 P1 安装指引，不擅自开拍。
5. 不选择 render runtime（遵守 AGENT_GUIDE HARD RULE；P0 只报告可用性）。

内容来源：压缩移植 `skills/meta/onboarding.md`，删除 monorepo 专用 bash，改为 MCP 调用。

### 5.2 `skills_eclaw/openmontage-gates-intro/SKILL.md`

```yaml
---
name: openmontage-gates-intro
description: Explain OpenMontage human approval gates and how users should reply.
metadata:
  openclaw:
    os: [win32, darwin, linux]
    emoji: "🚦"
---
```

正文覆盖：

- proposal / script / scene_plan / assets 关卡含义；
- 推荐回复：「选 B」「脚本通过」「分镜通过」「试听通过」；
- 为何付费前必须停下；
- 成片路径：`OPENMONTAGE_PROJECTS_DIR/<id>/renders/final.mp4`（P1 才有）；
- **不含** reviewer 细则与 checkpoint 写入 API（P1）。

### 5.3 Agent 可见性配置片段

```json5
{
  skills: {
    load: {
      extraDirs: ["<OPENMONTAGE_REPO_ROOT>/skills_eclaw"],
      watch: true
    },
    entries: {
      "openmontage-router": { enabled: true },
      "openmontage-gates-intro": { enabled: true }
    }
  },
  agents: {
    defaults: {
      skills: ["openmontage-router", "openmontage-gates-intro"]
    }
  }
}
```

说明：安装文档同时提供「复制到 `~/.openclaw/skills`」与「`extraDirs` 指向仓库」两种方式；eClaw Skill 广场导入标为待核验。

## 6. 权限基线（P0）

来自 Guide/04 §8，落地为可复制模板 `docs/Mcp_Entity/Install/templates/p0-openclaw.policy.json5`（实现阶段创建）：

```json5
{
  tools: {
    profile: "messaging",
    allow: ["bundle-mcp", "openmontage-doctor__*"],
    deny: ["group:runtime", "browser", "gateway"]
  },
  agents: {
    defaults: {
      sandbox: {
        mode: "all",
        workspaceAccess: "none"
      },
      skills: ["openmontage-router", "openmontage-gates-intro"]
    }
  }
}
```

注意：

- 若 `init_project` 需要写盘且 sandbox `workspaceAccess: none`，P0 有两条路径（实现时二选一并写进安装说明）：
  - **A（推荐诊断默认）：** 默认 Agent 不允许 `init_project`；仅 doctor/list/validate；建项目留给 P1 或显式 elevated 配置。
  - **B：** 为「装机助手」单独 Agent：`workspaceAccess: rw` 且仅映射 `OPENMONTAGE_PROJECTS_DIR`。
- P0 验收以 **诊断 + 3 提示词** 为主；`init_project` 作为次级验收项。

## 9. 风险与缓解

| 风险 | 缓解 |
|------|------|
| eClaw 不读 `~/.openclaw` | OpenClaw 先验收；Install 文档双轨；差异表 |
| MCP 工具名前缀 `server__tool` | Skill 正文写「调用 doctor 工具（名称以宿主列出为准）」 |
| sandbox 导致 MCP 全隐 | 安装说明强制 `bundle-mcp` / `openmontage-doctor__*` |
| `import tools` 依赖 monorepo | P0 固定 `cwd`=repo root；后续再抽 core |
| Windows 编码 | `PYTHONUTF8=1`；doctor 输出 ASCII-safe 摘要字段 |
| `init_project` 与 sandbox none 冲突 | 默认 Agent 不做写盘；另设装机 Agent 或延后到 P1 |
