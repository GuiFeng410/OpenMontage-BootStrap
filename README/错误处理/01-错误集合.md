## 1. BGM 素材无声（-91dB ≈ "不存在"）

**阶段**: 测试  
**现象**: 从 Pixabay 下的 MP3（clouds.mp3, piano.mp3）编码到 AAC 后只有约 2kbps，几乎静音。  
**根因**: `ffprobe loudnorm` 显示 `input_i: -inf`，`volumedetect` 为 `mean_volume: -91.0 dB`。第三方库返回了无效占位音频。  
**解决**: 用零 Key 合成 ambient BGM，128k AAC 编码。  
**教训**: Stock 音乐下载后必须先 `volumedetect` / loudnorm 验响度再混音。对应 Playbook **E01**。

## 2. FFmpeg 字幕滤镜在 Windows 路径上崩溃

**阶段**: 开发  
**现象**: `subtitles=D:\workfile\...\sub.srt` 报 `Unable to parse "original_size" option value`。  
**根因**: FFmpeg `subtitles` 滤镜把 `D:` 的冒号当成 `option:value` 分隔符。  
**解决**: 将 SRT 复制到无盘符相对路径（如 `assets/subs/_work/`）再烧录。  
**教训**: Windows 盘符路径勿直接塞进 FFmpeg filter。对应 Playbook **E02**。

## 3. PowerShell 的参数传递问题

**阶段**: 开发  
**现象**: 同一 FFmpeg 命令在 PowerShell 与 cmd/Python 行为不同（`&&` 截断、逗号被吃、引号嵌套失败）。  
**根因**: PowerShell 把 `&`、`,`、引号按自身语法解析，不是按 cmd/POSIX shell。  
**解决**: 全程 `subprocess.run([...], shell=False)` list 传参。  
**教训**: 媒体命令不要拼进 PowerShell 字符串。对应 Playbook **E03**。

## 4. FFmpeg `amix` 混音输出码率灾难

**阶段**: 开发  
**现象**: 长 WAV 混音后 AAC 仅 18–30KB，码率 2–4kbps。  
**根因**: 输入近静音时 AAC 自动降到最低码率。  
**解决**: 先出 WAV PCM，再单独 `-b:a 128k` 编码并验码率。  
**教训**: 混音与编码拆成可验证两步。对应 Playbook **E04**。

## 5. Agnes API 401：本地 Key 与有效 Key 不一致

**阶段**: 测试  
**现象**: `.env` 已填 `AGNES_API_KEY`，本机工具显示 `available`，但调用 `apihub.agnes-ai.com` 的 chat/image/video 均返回 **401 无效的令牌**；用户在 cmd 用 curl 同一账号 Key 却成功。  
**根因**:
1. `.env` 中的 Key 与 curl 验证成功的 Key **差 1 个字符**（粘贴/误改导致），服务端按「错误 Key」拒绝；
2. 过程中曾把 Key「纠正」成错误拼写，加剧不一致；
3. 本地只检查「变量非空」，**不能**等同于「云端鉴权通过」。  
**解决**:
1. 以用户 curl 成功的完整 Key 为准写回 `.env`（单行、无引号、无空格）；
2. 生成前用轻量 `POST /v1/chat/completions` 做鉴权探针（期望 HTTP 200）；
3. 勿根据 OCR/猜测改 Key。  
**教训**:
- Agnes **401** 优先查 Key 是否完整、是否与控制台/已验证 curl 一致，而不是先改业务代码；
- 接入新供应商时增加「鉴权探针」再跑付费/耗时视频任务；
- Agent 不得擅自改写用户密钥字符。

## 6. Agnes 视频任务完成但取不到成片 URL

**阶段**: 开发  
**现象**: Key 有效后，`agnes_video` 创建任务成功并轮询结束，却报 `Agnes video output missing metadata.url`，本地无 mp4。  
**根因**:
1. 文档示例把成片放在 `metadata.url`；
2. **实况 API** 完成态把下载地址放在**顶层** `url`，且常无 `metadata` 字段；
3. 状态机除 `queued` / `in_progress` / `completed` 外，还会出现 **`pending`**，旧逻辑只认文档状态集合。  
**解决**（已改 `tools/video/agnes_video.py`）:
1. 取 URL：`result.url` → `metadata.url` → `video_url` 依次回退；
2. 轮询：将 `pending` 视为进行中；失败态兼容 `failed` / `error` / `cancelled`；
3. 单测改为覆盖顶层 `url` 实况形状。  
**教训**:
- 供应商文档与实况响应可能不一致，**以一次真实 create→poll 的 JSON 字段为准**写解析；
- 异步视频工具应对「文档字段 + 实况字段」做兼容，并在缺失 URL 时打印可用 key 列表便于排查；
- 样片宜先 3–5 秒验证链路，再拉长时长。
