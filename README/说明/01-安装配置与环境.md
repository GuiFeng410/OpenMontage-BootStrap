# 01 — 安装、配置与环境检测

> 前提：已有 OpenClaw。细节由 Skill 执行；本文只给入口与清单。

## 安装闭环（推荐顺序）

```text
① OpenClaw 先放入并启用「安装 Skill」installer（外置拷贝一份即可）
② 对 Agent 说「按 BootStrap 安装」→ 选本地目录 <REPO> → clone（GitHub 优先，失败用 Gitee）
③ 仍由 installer 口述：venv → 注册 4 个 MCP → extraDirs 指向仓内 skills → 启用 Skill
④ 闭环检查通过 → setup 检测依赖 → verify_ready → 可出片
```

仓内 Skill **靠 extraDirs 同步**（`git pull` 即更新），不必把 setup/produce 拷进 `~/.openclaw/skills`。

## 对应 Skill

| 场景 | Skill |
|------|-------|
| 还没有本仓库 | `openmontage-bootstrap-installer`（先拷到 OpenClaw 用户 Skill 目录） |
| 环境检测 / 装依赖 | `openmontage-bootstrap-setup` |
| 出片 | `openmontage-bootstrap-produce`（见 [02](./02-免费与收费能力.md)） |
| 工具失败修复 | `openmontage-bootstrap-error-handling`（强烈建议） |
| 字幕 / 本地 BGM | `openmontage-bootstrap-captions-music`（强烈建议） |

模板与字段：[../配置/](../配置/)

## B. 注册 MCP / 启用 Skill（口述，不自动改配置）

**默认一并注册 4 个 MCP**（Key 可空；模板见 [配置/templates/](../配置/templates/)）：

| Server | 模板 |
|--------|------|
| `openmontage-bootstrap` | `bootstrap.mcp.json` |
| `openmontage-providers-tts` | `providers-tts.mcp.json` |
| `openmontage-providers-image` | `providers-image.mcp.json` |
| `openmontage-providers-video` | `providers-video.mcp.json` |

**extraDirs：** `<REPO>/openmontage/skills`

**必开 Skill：**

- `openmontage-bootstrap-setup`
- `openmontage-bootstrap-produce`
- `openmontage-bootstrap-providers`

**强烈建议一并启用：**

- `openmontage-bootstrap-error-handling`（失败：capture → plan → **apply**，含零 Key 合成 BGM）
- `openmontage-bootstrap-captions-music`（文稿字幕 + 本地 BGM）

Stock / 付费执行 Skill 按需再开，见 [02](./02-免费与收费能力.md)、[03](./03-字幕与配乐.md)。

易错字段摘要见 [配置/00-字段清单.md](../配置/00-字段清单.md)；逐项验收口令见 installer Skill。

## C. 环境检测与安装（setup）

1. `detect_environment` → 看缺口
2. `plan_install` → **先展示计划，勿直接装**
3. 你确认后：`dry_run=false` + `confirm_execute=true` 逐步执行
4. 若返回 `manual_commands`：交给你本机执行
5. `verify_ready` 为真后再出片

示例：

> 先检测环境，给我看完整安装计划，不要直接改系统。

---

## D. 已装机器只补新能力（已经拉取过项目，后续想要自己更新补充MCP/SKILL）

1. `git pull`（bootstrap 或 gitee `main`）— Skill 经 extraDirs 自动变新
2. OpenClaw **只新增** MCP/Skill（模板在配置/），不改动已有门面项
3. Key 用到再填；不填仍可轻度零 Key 出片
